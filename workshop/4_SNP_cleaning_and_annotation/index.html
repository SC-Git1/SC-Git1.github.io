4.1 SNP annotation with snpeff

SNP annotation with snpeff

Run the vt commands to clean up the VCF file. 

cat raw_snps.vcf \
| vt decompose - \
| vt normalize -n -r hg38_chr22.fa - \
| vt uniq - \
| vt view -f "QUAL>30" -h - \
> clean_snps.vcf

First, vt decompose turns multi-allelic variants into bi-allelic variants. According to the output, 11 variants are decomposed. Next, vt normalize converts the variants into a standardized format and vt uniq drops any duplicates. Finally, vt view filters out the SNPs with a QUAL value below your desired cutoff and the remaining SNPs are written to a file "clean_snps.vcf".

4.2 SNP annotation with snpeff

Now that we have a list of cleaned SNPs, we can annotate them to predict their effect using snpeff. The prgram snpeff is distributed as a .jar file that can be run with java. First, define the location of the .jar file. If you followed the conda environment set-up, the file should be located in the share/snpeff-5.2-0 folder of your conda environment. We will also store the current directory in a DIR variable. 

SNPEFF_JAR="${VSC_DATA}/miniconda3/envs/ws_snp_calling/share/snpeff-5.2-0/snpEff.jar"
DIR=$(pwd)
Now, we can run snpeff, using: 
java -jar ${SNPEFF_JAR} eff hg38 -dataDir $DIR \
clean_snps.vcf > annotated_snps.vcf
The command instructs to run the snpeff file using java, annotate the SNPs using the hg38 genome (which will be downloaded) and store the output in the current directory. Have a look at the generated files. First, snpEff_summary.html contains statistics of the SNP annotation. Details for each SNP were written to annotated_snps.vcf. Finally, an overview of the affected genes are provided in the file snpEff_genes.txt.

4.2.1 Adapt snpEff setting to your organism

Before running snpEff on another organism than H. sapiens, you should set the genome and genetic code table. Detailed instruction are provided at in the official documentation: https://genomics.sschmeier.com/ngs-voi/#prepare-snpeff-database
