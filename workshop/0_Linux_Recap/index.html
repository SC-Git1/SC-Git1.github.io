Introduction

This module contains Linux commands and terminology that will be essential during the workshop. Feel free to go through the module. You can also use it as a handy reference to circle back to anytime you need a refresher.

Glossary

This glossary contains terms that will bbe used throughout the workshop. Note that some terms may have a different definition depending on the context. The definitions here reflect their meaning in the context of Linux terminals.

TermDefinitionExample
ArgumentAn argument is input given by the user in a command. dir_test (in cd dir_test)
CommandA command is an instruction given by a user to perform a specific task. The first word of every command is called the command name. pwd, ls -l, cd
Command Line Tool (CLI)A command line tool is a text interface to your computer. The CLI takes in commands, and passes them on to the shell. The shell interprets the command and then passes them on to the operating system to run.
Command NameA command name is the first word of every command. pwd, ls, cd
Graphical User Interface (GUI)A graphical user interface is a visual way to interact with a computer using items like windows, icons, and menus.
KernelThe kernel is the heart of the OS. For example, Windows has the NT kernel, and macOS has the XNU kernel. NT kernel
(of Windows),
XNU kernel
(of macOS)
LinuxTechnically, Linux is a kernel. However, when most people say “Linux”, they are referring to a whole OS that uses the Linux kernel. There are many OS that use Linux, and they are called distributions. Ubuntu is one example of a Linux distribution.
Operating systemThe operating system is a software that serves as an intermediary between the user and the computer hardware. It allows users to interact with the computer without needing to understand the computer's language. It performs fundamental tasks like management of the file system, management of resources such as memory and processes, and handling input and output. It consist of a kernel, a shell and other programs that deal with user interface, networking etc. Windows, macOS, Ubuntu
Option An option is an argument that changes the behavior of a command. It is a predefined string that usually starts with a single or a double dash ("-" or "--"). -l (in ls -l)
PathA path is the location of a file or a directory in a human-readable format./home/usr2
Prompt A prompt is a small section of text that appears on the screen when a user opens a terminal. user@host:~$
ShellA shell is a program that processes and executes commands and prints output. bash
Standard OutputThe standard output is the default place to where a program sends its output data. By default, this is the terminal screen.
TerminalA terminal is an example of a command line interface. It is a wrapper program which runs the shell.

File system

The Linux file system is a structured hierarchy of files and directories. The starting point of the hierarchy is called the root and is represented by a single forward slash: /. Files or empty directories form the endpoints of the hierarchy. You can refer to an item in a Linux file system by its path. A path can be absolute or relative. In the Linux file system, an absolute path starts from the root directory "/", followed by the full path to the item. An example is /home/usr2 in the figure below. Relative paths start from the current directory. If for example you are in /home, your can type "usr2" or "./usr2". If you need to go up a directory level i.e., start the relative path from the parent directory, you can start the filepath with "../".

A Linux file system. The root is indicated by a forward slash. Three subdirectories of the root are shown here: /bin, /home, and /lib. The directory of user 2 can be found under /home/usr2. The relative path from /home to the directory of user 2 is "usr2" or "./usr2". the relative path from /home to the root directory is "..".

List of Linux commands

Moving forward, we will go over a set of Linux commands that you will find yourself using quite often throughout this workshop. These commands are essential tools in your Linux toolkit. You can follow the code in your terminal. Note that you can find the manual page of each command by typing the command name followed by --help.

pwd

The command pwd stands for print working directory. It prints the full filename of the current working directory

pwd

ls

The command ls is used to list the contents of a directory (by default the current working directory). To list the contents of a different directory, you can add the directory's filename as an argument. Options include -l to use the long listing format, -h to print sizes in human readable format and
-a to also show entries that start with "." (typically the current and parent directory). Options can be combined e.g., -lah. To see the full list of options, you can type ls --help into a terminal. 

ls
ls -lah
ls  /path/to/directory
ls -lah  /path/to/directory

cd

If you want to change the working directory you can use the command cd. It stands for change directory and can be used in the following way: 

cd /path/to/directory
cd ./path/to/directory
cd ../path/to/directory

mkdir

The command mkdir stands for make directory. As the name suggests, it is used to create new, empty directories. Note that if the directory already exists, mkdir will throw an error. The command is used in the following way: 

mkdir /path/to/directory

An option of mkdir that is used often is -p. As an exercise, try to find the meaning of this option in the manual page of the command using the terminal.

Check your answer

When you type mkdir --help into the terminal and press enter, the manual page appears. With the option -p, mkdir handles non-existing parent directories by creating them as needed, and it does not throw an error if the directory already exists.

cp

The command cp copies files or directories. To copy a file or an empty directory, use:

cp /path/to/item /path/to/copy

By default, only files are copied to the new location. Thus, if the directory you want to copy contains subdirectories, the command above will throw the error cp: -r not specified; omitting directory". As a solution, use the option -r to copy directories recursively.

mv

To mv a file instead of copying it, you can use the command mv. Note that you can also use mv to rename a file or directory. 
mv /path/to/item /new/path/to/item

rm

To remove a file, you can use the command rm. When you use rm on a directory, the command will throw an error rm: cannot remove '/path/tp/item': Is a directory. To remove a directory you should use the option -r.

Note that the error is thrown to help you. So, do not add the -r option unless you intend to remove a directory.

rm /path/to/file
rm -r /path/to/directory

cat

The command cat displays the entire content of a file to the standard output (by default).

cat /path/to/file

Note: always check how large the file is before displaying its entire content on the terminal screen. For large files it is adviced to use the commands more instead.

more

The command only the part of a file that fits the screen (= a page). On the last line, you can find the % of the file that has been viewed. To view the next line of the file, press "enter". To browse through the pages, press the upwards and downwards pointing arrow. To quit, press "q". When you have arrived at the end of the file, the command automatically quits.

more /path/to/file

head

To inspect the first lines of a file, you can use the command head. By default, this displays the first 5 lines. To change the number of lines on display, use the option -n, followed by an integer (int).

# to get everything except the header 
head /path/to/file
head -n int /path/to/directory

Note that similarly to head you can use tail to display the last lines of a file.
